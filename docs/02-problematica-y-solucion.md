# Problemática y solución propuesta

## 1. Problemática
Muchos negocios pequeños gestionan sus productos, clientes y ventas de forma manual o con herramientas dispersas.  
Esto genera problemas como:

- dificultad para saber el stock real disponible
- errores al registrar ventas
- duplicidad de información
- poca trazabilidad de clientes
- cálculos manuales incorrectos
- pérdida de tiempo al buscar información
- dificultad para consolidar datos para la toma de decisiones

En un contexto académico, esta problemática es adecuada porque permite modelar un caso real de organización y convertirlo en una solución incremental usando Programación Orientada a Objetos.

## 2. Situación actual
La tienda registra información en cuadernos, hojas sueltas o archivos no estructurados.  
Cada vez que se vende un producto:
- se revisa stock manualmente
- se calcula el total sin automatización
- no siempre queda registro del cliente
- no existe un historial claro de ventas

Esto ocasiona desorden operativo y riesgo de errores.

## 3. Consecuencias del problema
- productos sin control de stock
- ventas mal calculadas
- clientes mal registrados o no registrados
- pérdida de información
- procesos lentos
- escasa confiabilidad de los datos

## 4. Resolución propuesta
Se propone desarrollar **InventarioPro**, un sistema de gestión de inventario y ventas que centralice la información y automatice las operaciones básicas del negocio.

La solución permitirá:
- registrar y consultar productos
- registrar y consultar clientes
- registrar ventas con múltiples productos
- validar stock disponible antes de vender
- calcular subtotales y totales automáticamente
- mantener datos organizados en memoria y luego en base de datos
- disponer de una interfaz web básica para operar el sistema

## 5. Justificación técnica
El proyecto resuelve la problemática y, al mismo tiempo, permite aplicar directamente los contenidos del curso:

- **Clases y objetos:** modelan productos, clientes, ventas y usuarios.
- **Métodos y constructores:** definen el comportamiento y la inicialización de las entidades.
- **Estructuras de control:** permiten validar condiciones, ciclos y reglas de negocio.
- **Math y String:** ayudan en cálculos y validación/manipulación de texto.
- **Colecciones:** almacenan datos y soportan búsquedas y ordenamientos.
- **Herencia y polimorfismo:** mejoran el diseño del sistema.
- **Persistencia:** permite guardar la información para uso real.
- **JSP/Tomcat:** facilitan una interfaz web básica.

## 6. Beneficios esperados
- mejor organización de la información
- reducción de errores manuales
- mayor rapidez en el registro de operaciones
- trazabilidad de productos, clientes y ventas
- mejor experiencia de aprendizaje porque el estudiante construye un sistema completo

## 7. Resultado esperado
El resultado esperado es una aplicación académica funcional, coherente y escalable, desarrollada progresivamente durante el curso, que evidencie el dominio de Programación Orientada a Objetos aplicada a un caso real.

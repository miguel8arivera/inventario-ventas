# Alcance del proyecto

## 1. Nombre del proyecto
**InventarioPro - Sistema de Gestión de Inventario y Ventas**

## 2. Propósito
Desarrollar un sistema académico funcional que permita aplicar progresivamente los temas del curso Programación Orientada a Objetos mediante un caso real de negocio: el control de inventario, clientes y ventas de una tienda.

## 3. Objetivo general
Construir una aplicación en Java con interfaz web básica y persistencia de datos que demuestre el dominio de los principales temas del curso.

## 4. Objetivos específicos
- Modelar entidades del dominio usando clases y objetos.
- Implementar atributos, constructores y métodos.
- Aplicar estructuras de control a reglas del negocio.
- Usar clases nativas `Math` y `String` para cálculos y validaciones.
- Utilizar colecciones para almacenar y procesar datos.
- Aplicar herencia, encapsulamiento y polimorfismo.
- Integrar una interfaz web básica con JSP y Apache Tomcat.
- Persistir datos usando JDBC o JPA.

## 5. Alcance funcional
El sistema incluirá las siguientes funcionalidades:

### 5.1 Gestión de productos
- Registrar productos.
- Editar productos.
- Eliminar o desactivar productos.
- Consultar listado de productos.
- Buscar productos por código o nombre.
- Mostrar stock disponible.

### 5.2 Gestión de clientes
- Registrar clientes.
- Editar datos de clientes.
- Consultar clientes registrados.
- Buscar clientes por documento o nombre.

### 5.3 Gestión de ventas
- Registrar ventas.
- Agregar uno o varios productos a una venta.
- Calcular subtotal y total.
- Validar stock antes de confirmar la venta.
- Emitir resumen de la venta.

### 5.4 Reportes básicos
- Listar ventas realizadas.
- Consultar productos con bajo stock.
- Consultar clientes registrados.
- Ver resumen de ventas.

## 6. Alcance técnico
El proyecto debe demostrar la aplicación de los siguientes temas del curso:
- clases y objetos
- atributos locales y globales
- constructores
- métodos de clase y de instancia
- estructuras selectivas y repetitivas
- clases nativas `Math` y `String`
- colecciones y genéricos
- herencia
- encapsulamiento
- polimorfismo
- persistencia de datos con Java
- interfaz básica con JSP/Tomcat

## 7. Alcance académico
Este proyecto está diseñado para desarrollarse durante todo el ciclo y crecer sesión a sesión hasta convertirse en el producto final del curso.

## 8. Fuera del alcance
En esta versión no se considera:
- integración con pagos en línea
- facturación electrónica oficial
- seguridad avanzada
- control de roles complejo
- API REST
- despliegue en nube
- analítica avanzada
- app móvil nativa

## 9. Entregable final esperado
Al finalizar el curso se espera un sistema que:
- funcione de forma básica y coherente
- tenga interfaz web navegable
- maneje productos, clientes y ventas
- aplique principios de POO
- almacene datos en base de datos
- sea sustentable como proyecto integrador del curso
